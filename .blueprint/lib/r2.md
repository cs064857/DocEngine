# lib/r2.ts

## 職責契約
此模組是 Cloudflare R2 的統一儲存適配層，封裝 S3 相容 client 建立、預設/覆蓋憑證解析、物件讀寫，以及任務狀態 JSON 的持久化約定。它的目的在於把儲存細節從 API 路由與處理流程中抽離；嚴禁承擔 HTTP 協議處理、URL 到物件 key 的命名策略、內容清洗、抓取流程控制或佇列編排。

## 接口摘要
### `JobTask`
- **資料形狀**：`taskId`、`status`、`total`、`completed`、`failed`、`failedUrls`、`retryingUrls?`、`date`。
- **用途**：定義任務狀態在 R2 JSON 檔中的共用契約。

### `R2Overrides`
- **資料形狀**：`accountId?`、`accessKeyId?`、`secretAccessKey?`、`bucketName?`。
- **用途**：允許呼叫端以 request-scoped 方式覆蓋部署預設 R2 設定。

### `putObject(key, content, contentType?, r2?)`
- **輸入**：物件 key、字串或 Buffer 內容、MIME 類型、可選 R2 覆蓋值。
- **輸出**：`Promise<void>`。
- **副作用**：將內容寫入解析後的 R2 bucket。
- **約束**：必須能從覆蓋值或 `config.r2` 取得完整憑證。

### `getObject(key, r2?)`
- **輸入**：物件 key、可選 R2 覆蓋值。
- **輸出**：UTF-8 字串內容。
- **副作用**：自 R2 下載物件並轉為字串。
- **約束**：空內容或缺檔視為錯誤，而非靜默回傳空值。

### `listObjects(prefix?, limit?, r2?)`
- **輸入**：可選前綴、最大筆數、可選 R2 覆蓋值。
- **輸出**：R2 物件中繼資料陣列。
- **副作用**：向 bucket 發出列舉請求。

### `putTaskStatus(taskId, statusObj, r2?)`
- **輸入**：任務 ID、`JobTask` 狀態物件、可選 R2 覆蓋值。
- **輸出**：`Promise<void>`。
- **副作用**：以 `tasks/{taskId}.json` 規約寫入任務狀態。

### `getTaskStatus(taskId, r2?)`
- **輸入**：任務 ID、可選 R2 覆蓋值。
- **輸出**：`JobTask | null`。
- **副作用**：讀取並解析任務狀態 JSON。
- **約束**：找不到 key 時正規化為 `null`；其他儲存錯誤向上拋出。

## 依賴拓撲
`lib/config.ts` → `lib/r2.ts` →
- `app/api/files/route.ts`：提供檔案讀取與列舉。
- `app/api/tasks/route.ts`、`app/api/status/[taskId]/route.ts`：提供任務狀態查詢。
- `app/api/crawl/route.ts`：初始化任務狀態檔。
- `app/api/queues/process-url/route.ts`：寫入 raw/cleaned 內容並持續更新任務狀態。
- `app/api/scrape/route.ts`：保存即時 scrape 產物。

- 在本 bundle 內，`lib/r2.ts` 是核心儲存脊柱：`lib/config.ts` 提供預設設定，`app/api/files/route.ts` 則將其讀取能力轉為 HTTP 介面。
- `lib/utils/helpers.ts` 不直接被此模組呼叫，但它生成的 taskId、日期與 R2 key 命名規則，是多數呼叫端傳入本模組的上游輸入。
